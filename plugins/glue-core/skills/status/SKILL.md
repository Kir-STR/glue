Проверь состояние доставки правил Glue, выполнив в Bash:

```bash
node "${CLAUDE_PLUGIN_ROOT}/bin/glue.mjs" status
```

Команда выводит JSON `{ mode, missing[], changed[], stale[], packs[], summary }`. Разбери его и доложи оператору на понятном языке:

**Формат вердикта:**

1. **Итог:** строка из поля `summary` — вывести как есть, она уже человекочитаема (пример: «native delivery active: 5 files» или «fallback: 2 file(s) missing»).

2. **Детали по проблемным файлам** (если есть):
   - `missing` непуст → перечислить пути отсутствующих файлов: «Отсутствуют: …»
   - `changed` непуст → перечислить пути изменённых файлов: «Изменены (хэш не совпадает): …»
   - `stale` непуст → перечислить пути устаревших файлов: «Устарели (версия пака изменилась): …»
   - Если все три пусты и mode = `'native'` — подтвердить: «Все файлы на месте, хэши совпадают, версии актуальны.»

3. **Обнаруженные паки:** из поля `packs` — перечислить `name@version`. Если `packs` пуст — сообщить: «Контент-паки не обнаружены.»

4. **Рекомендация** (только при mode = `'fallback'`): доставка правил идёт через fallback-инъекцию (хук вставляет тела правил в контекст). Для активации нативной доставки — выполнить `/glue-core:init` и затем `/clear` или начать новый сеанс.
